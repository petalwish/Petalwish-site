# Vibe Coding Prompt: The Sequence Decoder Scroll Guide

## 1. 视觉效果描述 (Visual Storytelling)
逻辑： 滚动不再是“拉开”序幕，而是“稳定”数据流。

视觉表现：在屏幕底部中央，有一行不断高速跳动的随机字符流（A, T, C, G 与 0, 1 混合）。

滚动反馈：当你向下滚动时，这些字符会从左至右逐个“锁定”并停止跳动，最终拼凑出指令：EXECUTE_LAB_CLUSTERS.SH
- **核心意象**：模拟生物信息学中的序列比对（Sequence Alignment）与实时解码过程。
- **初始状态**：屏幕底部中央显示一行正在高速跳动、闪烁的随机字符流，包含 `A, T, C, G` 与 `0, 1`。字体使用 `JetBrains Mono`，颜色为半透明的 `Slate-400` (#94A3B8)。
- **滚动交互**：
    - 随着用户向下滚动，字符流从左至右逐个停止跳动并“锁定”为正确字符。
    - 锁定后的字符颜色由 `Slate-400` 变为具有发光感的 `Cyan-400` (#22D3EE)。
    - 字符锁定的进度与滚动条位置（scroll progress 0% - 100%）严格线性对应。
- **最终状态**：当滚动接近 Hero Section 底部时，字符流完全固定为：`>> EXECUTE_LAB_CLUSTERS.SH`。
- **整体氛围**：极简、硬核、具有“北极冷光”实验室的精密感。

## 2. 逻辑实现伪代码 (Logic Pseudocode)
```typescript
/**
 * 序列译码器核心逻辑
 */
const TARGET_COMMAND = ">> EXECUTE_LAB_CLUSTERS.SH";
const GLITCH_CHARS = "ATCG01";

// 1. 获取滚动进度 (0 到 1)
const scrollProgress = window.scrollY / window.innerHeight;

// 2. 计算当前应锁定的字符数量
const charsToLock = Math.floor(scrollProgress * TARGET_COMMAND.length);

// 3. 渲染每一帧
function renderFrame() {
  let displayString = "";
  
  for (let i = 0; i < TARGET_COMMAND.length; i++) {
    if (i < charsToLock) {
      // 已锁定部分：显示目标字符，应用 Cyan-400 样式
      displayString += `<span class="text-primary-cyan">${TARGET_COMMAND[i]}</span>`;
    } else {
      // 未锁定部分：每帧随机切换 GLITCH_CHARS
      const randomChar = GLITCH_CHARS[Math.floor(Math.random() * GLITCH_CHARS.length)];
      displayString += `<span class="text-slate-400 opacity-50">${randomChar}</span>`;
    }
  }
  
  decoderElement.innerHTML = displayString;
}

Vibe Coding 交付要求 (Success Criteria)

动画曲线：锁定动作要清脆利落，消散动作遵循 transition: all 1.2s cubic-bezier(0.23, 1, 0.32, 1)。

响应式控制：在移动端适当减少字符跳动频率以节省电力。

视觉细节：

锁定后的字符需带有微弱的 drop-shadow 霓虹光晕。

在字符流下方保留一个极细（0.5px）的进度条，随滚动进度同步增长。

集成提示：将此组件放置在 Hero Section 的 bottom-8 位置，确保其 z-index 高于背景的 DNA 3D 点云，但低于全局导航栏。
